CERES API

Introduction

This document describes the API for the Cloud Enterprise Rapid and Easy Storage application. There are five types of requests: collect, modify, delete, query, and metric. Collect is used for storing data into the database, Modify is used to update information in the database, and delete is used to remove information from the database. Query is used to retrieve raw information from the database and Metric is used to obtain metrics from the database, including min/max, mean, median, sum, and variance/co-variance.

All requests are HTTP requests: Collect, Modify, and Delete are POST, Query and metric are GET. Each POST request has a header and a body (which is always a JSON object). Each GET request has a single url parameter called q which is a URL encoded JSON object. The sections below describe the headers and bodies for the different types of requests below. The responses from the application (what a client application should expect to receive) are described after the requests.

Collect

Collect allows a client to add data to the database. Collect is a POST request to the /collect URL of the server.

Most of the information in the header relates to OAuth. This information will be in the header of all requests, not just Collect. Currently, this application only makes use of the HMAC signature portion of OAuth and the other OAuth parameters are left blank. In the future, the application may take full advantage of OAuth. The example OAuth parameters were taken from the OAuth spec.

Modify

Modify allows updating of the data in the database. In the future, it may require some sort of administrative priviledges to be used. It is a POST request to the /modify URL of the server.

Header

			
POST /modify HTTP/1.1
Host: www.example.com

Authorization: OAuth realm="Example",
    oauth_consumer_key="",
	oauth_token="",
	oauth_signature_method="HMAC-SHA1",
	oauth_signature="wOJIO9A2W5mFwDgiDvZbTSMK%2FPY%3D",
	oauth_timestamp="137131200",
	oauth_nonce="4572616e48616d6d65724c61686176",
	oauth_version="1.0"

Body

			
{
  "type" : "modify",
  "time_utc" : 1350491137,
  "authorize_id" : 0,
  "expr" : {
        "name.first": "John",
        "name.last": "Backus"
    },
  "data" : {}
}

The body is a JSON string with five required name/value pairs: type, time_utc, authorize_id, expr, and data. These are described in Table 2. See the Query command for more information on how to use the expr parameter.

Table 2. Parameters for modify

ParameterTypeDescription
typestringset to "modify"
time_utcnumberthe unix epoch time when the updated data was generated
authorize_idnumberset to 0 (for now). may be used to identify users later
exprarraycontaining expressions of the values to update in the database.
dataobjectthe updated data

Delete

Delete allows removal of the data in the database. In the future, it may require some sort of administrative priviledges to be used. It is a POST request to the /delete URL of the server.

Header

			
POST /delete HTTP/1.1
Host: www.example.com

Authorization: OAuth realm="Example",
    oauth_consumer_key="",
	oauth_token="",
	oauth_signature_method="HMAC-SHA1",
	oauth_signature="wOJIO9A2W5mFwDgiDvZbTSMK%2FPY%3D",
	oauth_timestamp="137131200",
	oauth_nonce="4572616e48616d6d65724c61686176",
	oauth_version="1.0"

Body

			
{
  "type" : "delete",
  "authorize_id" : 0,
  "expr" : 
    {
        "name.first": "John",
        "name.last": "Backus"
    },
  "justOne" true
}

The body is a JSON string with four required name/value pairs: type, authorize_id, expr, and justOne. These are described in Table 3.

Table 3. Parameters for delete

ParameterTypeDescription
typestringset to "delete"
authorize_idnumberset to 0 (for now). may be used to identify users later
exprarraycontaining expressions of the values to delete in the database.
justOnebooleanif true, delete just the first record that matches, if false, delete ALL records that match

Metric

Metric allows a client to obtain aggregate information on the collected data. Metric is a GET request to the /metric URL of the application.

Header

				
GET /metric?q=<q parameter> HTTP/1.1
Host: www.example.com

Authorization: OAuth realm="Example",
    oauth_consumer_key="",
	oauth_token="",
	oauth_signature_method="HMAC-SHA1",
	oauth_signature="wOJIO9A2W5mFwDgiDvZbTSMK%2FPY%3D",
	oauth_timestamp="137131200",
	oauth_nonce="4572616e48616d6d65724c61686176",
	oauth_version="1.0"

q parameter

		
{
	"type" : "metric",
	"authorize_id" : 0,
	"expr" : "sum(total)",
	"filter" : [
	"is(provider, 'trader joes')",
	"between(time_utc, 1350401137, 1350491137)"
	]
}

The q URL parameter is a URL-encoded JSON string with four required name/value pairs: type, authorize_id, expr, and filter. These are described in Table 5. See Table 6 for the types of metrics.

Body

The response body will consist of a single JSON object in the following format: 

			
{
	"code" : 0,
	"message" : "message text",
	"data" : {},
}

The code will be one of the integers listed in the codes table, and the message will be a human-readable string describing what happened during the request. The data will be a free-form JSON object containing any data that is returned from the query. It will be empty for the Collect and Modify requests.
